Recuperação de dados (tabela) apagados sem backup

Se acidentalmente foi executado um comando DROP em um objeto e tem-se um backup full do banco, considerando que nenhuma alteração na tabela foi feita após a exclusão, basta recuperar os dados sobrescrevendo o banco original com o backup. Porém, se este não é o caso, ou por que o objeto e os dados foram modificados após o drop no SQL executado ou por falta de backup do banco, há um meio de recuperar os dados perdidos.

Cada operação, incluindo drop, são registradas no transaction log, o qual armazena informações necessárias para reverter a transação e, uma vez que o banco de dados está em modo de recuperação full, suporta uma recuperação point-in-tie de uma operação drop.

Quando o objeto é deletado e se o usuário der um comando SELECT no mesmo, uma mensagem de erro será exibida, indicando a não existência. Entretanto, há uma ferramenta auxiliar do SQL Server que recupera dados apagados, truncados, corrompidos ou danificados, recuperando objetos e dados perdidos devido a operação drop e restaura BLOBs ou como arquivos online.

Esta ferramenta/programa é chamado de ApexSQL Recover, bastando seguir os passos seguintes em casos de operações drop sem backup:

  1. Iniciar o ApexSQL Recover

  2. Selecione a opção recover lost data due to a drop operation

  3. Em Select the database to recover, execute:

    1. Especifique a instância na lista suspensa Server

    2. Especifique o método de autenticação para a instância.

    3. Especifique o nome da database que contêm as tabelas apagadas.

  4. No passo are additional data sources avaible?, selecione a opção no additional transaction logs are avaible

  5. No passo select a recovery action, selecione a opção savr recovery script to file e especifique o caminho onde será gravado o script T-SQL que irá inserir todos os dados perdidos quando executado

  6. Clique em recover
