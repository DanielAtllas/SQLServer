Introdução - SQL Server

  • O que é?

    • O SQL Server é um SGBD (Sistema Gerenciador de Banco de Dados) que auxilia na manipulação de todo um banco de dados, de forma possível a construir um banco do zero e monitorá-lo, modifica-lo e administra-lo de acordo com que novos dados sejam registrados.

    Este SGBD foi criado e é constantemebte atualizado pela empresa Microsoft, a qual disponibiliza este sistema de forma gratuita para seus usuários. Assim como, o SQL Server é um dos SGBD's mais utilizados pelas empresas, sendo em muitas das vezes, o core das instituições que o carregam.

    O SQL Server, como o nome já indica, utiliza de instruções SQL para sua manipulação, linguagem essa usada no gerenciamento de bancos de dados. Porém, por existirem inúmeros outros SGBD's, o SQL Server, além da linguagem SQL, possui também sua linguagem própria, chamada de T-SQL (Transact SQL), o qual usufrui de comandos próprios que só são lidos pelo SQL Server. Além disso, o SQL Server é funcional para a manipulação de bancos relacionais, que são bancos de dados constituídos de tabelas, colunas e tuplas.

  • Como instalar?

    • Para utilizar do SQL Server, é preciso que o usuário instale , primeiramente, alguns softwares em sua máquina, sendo eles essênciais para o bom uso do SGBD. Os programas necessários são:

    1. SSMS

      2. O SSMS ou SQL Server Management Studio, é um ambiente integrado para gerenciar qualquer infraestrutura de SQL do SQL Server

      O mesmo fornece ferramentas de configuração, monitoramento e administração do SQL Server e bancos de dados, podendo também possuir elementos de construção, consultas e scripts dos dados. Além disso, o SSMS é inteiramente gratuíto.


    2. SQL Server

      3. Nada mais é do que a integração do SQL Server em um sistema gerenciador de dados, sendo então utilizado pelo SSMS, neste caso.

  • Conectando-se a uma instância

    • Após a instalação do SSMS e do SQL Server serem realizadas em sua máquina, é preciso executar o software SSMS para a inicialização. Após isso, o programa será aberto e uma pequena janela será exibida, solicitando registros de Server name e Authentication. Esta janela é a conexão direta com uma instância.

    • Server Name

      • Este é o campo de Nome do servidor, sendo preciso inserir o hostname de sua máquina ou o nome da instância do SQL Server. Caso esteja usando o SQL Server de forma local, basta inserir o hostname de sua máquina, que o software irá interagir. Porém, se estiver utilizando-o em uma empresa, trabalho ou algo do tipo, é usado o nome da instância criada para tal funcionalidade.


    • Authentication

      • Este é o campo de Autenticação, sendo como padrão, utilizada a opção "Windows Authentication" (Também é possível utilizar a opção SQL Server Authentication. Porém, ao ser usada, deverá inserir um nome de usuário e uma senha).

    Feito isso, o processo de conexão com a instância estará concluído, onde um pequeno menu será exibido ao lado esquerdo do programa, este é chamado de Object Explorer, sendo basicamente um menu de atalho para conexões com instâncias, databases e suas tabelas, assim como a visualização dos mesmos. Portanto, o segundo passo, será a conexão (ou criação) com uma database.

  • Conectando-se a uma database

    • Para a conexão com uma database, 2 passos podem ser executados (através do object xplorer ou por T-SQL). A conexão com a database, irá dar ao usuário acesso aos dados diretos das tabelas presentes na mesma, assim como aos índices e outras funções.

    • Através do Object Explorer

      • Os passos a serem seguidos para a criação de uma database pelo object explorer é bem simples, bastando clicar com o botão direito em databases e selecionar a opção New Database, onde será aberta uma janela de configuração, pedindo para inserir um nome para a database. Depois basta apertar em OK e pronto, sua primeira database estará criada.

      Para se conectar a uma database já criada, o processo é semelhante, bastando clicar com o botão direito na database desejada e escolher a opção New Query e pronto, será aberta uma aba de inserção de Query's na database conectada.

    • Através do T-SQL

      • Também é possível criar uma database aravés de comandos T-SQL, de forma a serem dados através da aba de uma query. Ou seja, primeiramente, será preciso abrir uma aba de inserção de query na instância logada bastando apertar no botão "New Query" ou, basta apertar as teclas Ctrl+N que a aba será aberta.

      Com a aba aberta, insira o comando "CREATE DATABASE" seguido do nome que deseja da-la. desta forma, sua primeira database estará criada.

      Para se conectar a database criada, será preciso dar o mesmo comando de abertura de aba de query na database criada (Ctrl+N).

    OBS: Após criar a database, dê um atualizada na instância... Para fazer isso, basta apertar com o botão direito no nome da instância e clicar em Refresh.

    Feito os passos acima, seu banco de dados estará criado, bastando agora dar instruções SQL e T-SQL para administra-lo.
